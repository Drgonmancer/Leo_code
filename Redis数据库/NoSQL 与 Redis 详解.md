# NoSQL 与 Redis 详解

---

## 一、NoSQL 介绍

**NoSQL**（Not Only SQL）是一类新出现的数据库，泛指非关系型数据库。

### 主要特点

- 不支持 SQL 语法
- 存储结构与传统关系型数据库中的关系表完全不同
- NoSQL 中存储的数据都是 **KV 形式**
- 没有通用的语言，每种 NoSQL 数据库都有自己的 API 和语法

### 常见 NoSQL 产品

| 产品 | 所属阵营 |
|------|----------|
| Redis | 内存数据库 |
| MongoDB | 文档数据库 |
| HBase | Hadoop 分布式数据库 |
| Cassandra | Hadoop 分布式数据库 |

### NoSQL vs SQL 数据库

| 特性 | SQL 数据库 | NoSQL 数据库 |
|------|-------------|--------------|
| 适用场景 | 关系特别复杂的数据查询场景 | 简单查询场景 |
| 事务支持 | 支持非常完善 | 基本不支持 |
| 发展趋势 | 两者不断取长补短，呈现融合趋势 |

---

## 二、Redis 简介

**Redis** 是一个开源的使用 **ANSI C 语言**编写的、支持网络的、可基于内存亦可持久化的日志型 **Key-Value 数据库**，并提供多种语言的 API。

### 发展历程

- **2010年3月15日起**：Redis 的开发工作由 **VMware** 主持
- **2013年5月开始**：Redis 的开发由 **Pivotal** 赞助

### 定位

Redis 是 NoSQL 技术阵营中的一员，通过多种键值数据类型来适应不同场景下的存储需求，借助高层级接口可以胜任多种角色，如：

- 缓存系统
- 队列系统
- 其他应用场景

---

## 三、Redis 特性

1. **数据持久化**：支持将内存中的数据保存在磁盘中，重启后可再次加载使用
2. **丰富的数据类型**：不仅支持简单的 key-value 类型，还提供以下数据结构：
   - List（列表）
   - Set（集合）
   - Zset（有序集合）
   - Hash（哈希）
3. **数据备份**：支持 master-slave 模式的数据备份

---

## 四、Redis 优势

| 优势 | 说明 |
|------|------|
| **性能极高** | 读速度：110,000 次/秒，写速度：81,000 次/秒 |
| **丰富的数据类型** | 支持二进制案例的 Strings、Lists、Hashes、Sets 及 Ordered Sets 数据类型操作 |
| **原子性** | 所有操作都是原子性的 |
| **特性丰富** | 支持 publish/subscribe、通知、key 过期等等特性 |

---

## 五、Redis 应用场景

1. **缓存系统**：替代 ehcache/memcached，Redis 所有数据放在内存中（内存数据库）
2. **替代传统数据库**：在某些特定应用场景下，比如社交类应用
3. **大型系统特定功能**：
   - Session 共享
   - 购物车功能
4. **无限可能**：只要你有丰富的想象力，Redis 可以给你无限的惊喜！
