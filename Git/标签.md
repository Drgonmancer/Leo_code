# Git 标签管理指南

## 一、标签简介

| 作用 | 说明 |
|------|------|
| 记录大版本 | 标记重要的版本节点 |
| 备份大版本代码 | 保存特定版本的代码快照 |

> 💡 适用场景：当某个大版本开发完成后，需要打标签进行记录和备份

---

## 二、标签操作流程

### 步骤一：进入经理本地仓库

```bash
cd Desktop/manager/info/
```

### 步骤二：在本地打标签

```bash
git tag -a 标签名 -m '标签描述'

# 示例
git tag -a v1.0 -m 'version 1.0'
```

### 步骤三：推送标签到远程仓库

```bash
git push origin 标签名

# 示例
git push origin v1.0
```

### 步骤四：查看标签结果

```bash
# 查看所有标签
git tag

# 查看标签详情
git show 标签名
```

---

## 三、删除标签

| 操作 | 命令 |
|------|------|
| 删除本地标签 | `git tag -d 标签名` |
| 删除远程仓库标签 | `git push origin --delete tag 标签名` |

> ⚠️ 注意：删除操作需谨慎，远程标签删除后无法恢复

---

## 四、常用命令速查表

```bash
# 查看标签列表
git tag

# 创建标签
git tag -a v1.0 -m 'version 1.0'

# 推送单个标签到远程
git push origin v1.0

# 推送所有标签到远程
git push origin --tags

# 查看标签详情
git show v1.0

# 删除本地标签
git tag -d v1.0

# 删除远程标签
git push origin --delete tag v1.0

# 检出指定标签的代码
git checkout v1.0
```
