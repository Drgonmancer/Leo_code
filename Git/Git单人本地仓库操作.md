# Git单人本地仓库操作

> **课程目标**：学习常用的 Git 终端命令  
> **提示**：本地仓库是个 `.git` 隐藏文件

---

## 1. 安装 Git

```bash
sudo apt-get install git
# 密码: xxxx
```

## 2. 查看 Git 安装结果

```bash
git --version
```

## 3. 创建项目

在桌面创建 `test` 文件夹，表示工作项目。

## 4. 创建本地仓库

**4.1** 进入到 test 并创建本地仓库 `.git`

**4.2** 新创建的本地仓库是个空仓库

```bash
cd Desktop/test/
git init
```

**4.3** 创建后显示隐藏文件

进入待显示的文件路径，按 `Ctrl + H` 显示隐藏文件

---

## 5. 配置个人信息

```bash
# 设置用户名
git config user.name '张三'

# 设置邮箱
git config user.email 'zhangsan@163.com'
```

> **提示**：默认不配置会使用全局配置  
> 全局配置文件路径：`~/.gitconfig`

---

## 6. 新建文件

在项目文件夹 `test` 中创建 `login.py` 文件，用于版本控制演示。

---

## 7. 查看文件状态

```bash
git status
```

| 状态 | 含义 |
|------|------|
| 红色 | 新建/修改的文件，在**工作区** |
| 绿色 | 文件在**暂存区** |

> 新建的 `login.py` 文件在工作区，需要添加到暂存区并提交到仓库区

---

## 8. 将工作区文件添加到暂存区

```bash
# 添加项目中所有文件
git add .

# 添加指定文件
git add login.py
```

---

## 9. 将暂存区文件提交到仓库区

```bash
# -m 后面是版本描述信息
git commit -m '版本描述'
```

> `commit` 会生成一条版本记录

---

## 10. 编辑代码并提交

代码编辑完成后，添加和提交可以合并执行：

```bash
git commit -am "版本描述"
```

提交两次代码，会有两个版本记录。

---

## 11. 查看历史版本

```bash
# 查看当前分支的提交历史
git log

# 查看所有分支的所有操作记录（包括已删除的commit记录）
git reflog
```

> **区别**：`git reflog` 可以查看已删除的 commit 记录，`git log` 不能

---

## 12. 回退版本

### 方案一：使用 `HEAD` 指针

| 符号 | 含义 |
|------|------|
| `HEAD` | 当前最新版本 |
| `HEAD^` | 当前最新版本的前一个版本 |
| `HEAD^^` | 前两个版本 |
| `HEAD~n` | 当前最新版本的前 n 个版本 |

```bash
git reset --hard HEAD^
```

### 方案二：通过版本号回退

```bash
git reset --hard 版本号
```

> 当版本非常多时，可使用版本号回退到指定版本

---

## 13. 撤销修改

> 只能撤销工作区、暂存区的代码，不能撤销仓库区的代码。  
> 撤销仓库区的代码相当于回退版本操作。

### 撤销工作区代码

新加代码 `num3 = 30`，不 add 到暂存区，保留在工作区

```bash
git checkout 文件
```

---

## 14. 撤销暂存区代码

文件已添加到暂存区，但未提交到仓库区：

```bash
# 从暂存区撤销到工作区
git reset HEAD 文件名
```

---

## 常用命令速查表

| 命令 | 说明 |
|------|------|
| `git init` | 初始化本地仓库 |
| `git status` | 查看文件状态 |
| `git add .` | 添加所有文件到暂存区 |
| `git commit -m "描述"` | 提交到仓库区 |
| `git log` | 查看提交历史 |
| `git reflog` | 查看所有操作记录 |
| `git reset --hard 版本` | 回退到指定版本 |
| `git checkout 文件` | 撤销工作区修改 |
