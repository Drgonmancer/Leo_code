### Git 介绍

#### Git 的定义
Git 是目前世界上最先进的分布式版本控制系统（没有之一）。

#### 作用
- **源代码管理**

#### 为什么要进行源代码管理？
- **方便多人协同开发**
- **方便版本控制**

#### Git 的诞生
- **Linux 之父**: Linus Benedict Torvalds
- **开发目的**: 当初开发 Git 仅仅是为辅助 Linux 内核的开发（管理源代码）

### Git 工作流程图解析

#### 1. 用户机

##### 工作区 (Workspace)
- 在 IDE 中进行文件的添加、修改、删除操作。
- **核心操作**:
  - **文件添加或修改**:
    ```bash
    git add 文件名
    ```
  - **文件删除**:
    ```bash
    git rm 文件名
    ```
  - **文件恢复**:
    ```bash
    git checkout -- 文件名
    ```

##### 暂存区 (Index/Staging Area)
- 存储每天工作中的小片段更改。
- **核心操作**:
  - **提交更改到暂存区**:
    ```bash
    git add 文件名
    ```
  - **查看暂存区状态**:
    ```bash
    git status
    ```
  - **恢复到前一次提交状态**:
    ```bash
    git reset --hard
    ```

##### 版本库 (Repository)
- 存储本地的提交历史记录。
- **核心操作**:
  - **查看历史记录**:
    ```bash
    git reflog
    ```

##### 仓库区 (Local Branch)
- 存储用户分支上的工作，每个人在自己的分支上工作。
- **核心操作**:
  - **查看历史记录**:
    ```bash
    git reflog
    ```

##### 描述
